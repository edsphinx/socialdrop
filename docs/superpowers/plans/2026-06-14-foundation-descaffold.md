# Foundation Lock-in & De-scaffolding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `main` the verified, English, tested foundation and strip scaffold-eth-2 so SocialDrop reads as a professional product, with no behavior change.

**Architecture:** Two phases. Phase 0 verifies and merges the trustworthy `feature/grant-ready-cleanup` branch, makes the type-check green, rotates the exposed deployer key (with a Sepolia redeploy), and makes CI actually run tests + build. Phase 1 removes scaffold-eth on a `chore/descaffold` branch: migrate the `~~` alias to `@`, hand-author the contract ABI source, replace the wagmi/RainbowKit config with a clean `getDefaultConfig`, delete the dead/vendored scaffold trees, remove hackathon shortcuts, and rebrand. The existing 61 Vitest + 17 Hardhat tests plus a strict `next build` are the regression net run after every task.

**Tech Stack:** Next.js 15 (App Router), React 19, wagmi 2 / viem 2, RainbowKit 2, Prisma 6 / Supabase Postgres, Hardhat, Solidity 0.8.25, Vitest, Yarn 3 workspaces, scaffold-eth-2 (being removed).

**Conventions for this plan:**
- Run commands from the repo root `/Users/edsphinx/dev/blockchain/socialdrop/socialdrop` unless a `cd` is shown.
- This is a refactor/cleanup with no new behavior: the "test" after each task is `the existing suites still pass` + `the build/type-check is still green`, not a new failing test.
- Personal repo → push under the `edsphinx` GitHub identity over SSH (see global identity rules). Verify before pushing.
